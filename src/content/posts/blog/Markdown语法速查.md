# Markdown 语法速查

Markdown 是一种用纯文本写文档的标记语言，易读易写，广泛用于笔记、文档、博客和 GitHub 等平台。

---

## 一、基本语法

### 1. 标题

用 `#` 表示标题，`#` 数量对应级别（1–6 级）：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**效果：** 上面六行会显示为不同大小的标题。

---

### 2. 段落与换行

- **段落**：段落之间空一行。
- **换行**：行末加两个空格，或单独空一行（不同渲染器略有差异）。

```markdown
这是第一段。

这是第二段。
```

---

### 3. 强调（粗体、斜体）

| 写法                  | 效果              |
| ------------------- | --------------- |
| `*斜体*` 或 `_斜体_`     | *斜体*            |
| `**粗体**` 或 `__粗体__` | **粗体**          |
| `***粗斜体***`         | ***粗斜体***       |
| `~~删除线~~`           | ~~删除线~~（部分平台支持） |

---

### 4. 列表

**无序列表**（`-`、`*`、`+` 均可，注意符号后要有空格）：

```markdown
- 项目一
- 项目二
  - 子项目
```
- 1

**有序列表**：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

**任务列表**（GitHub、Obsidian 等支持）：

```markdown
- [ ] 未完成
- [x] 已完成
```

---

### 5. 链接与图片

**链接**：

```markdown
[显示文字](https://example.com)
[带标题的链接](https://example.com "鼠标悬停提示")
```

**图片**（语法与链接类似，前面加 `!`）：

```markdown
![替代文字](图片地址.png)
![替代文字](图片地址.png "图片标题")
```

**引用式链接**（正文更干净）：

```markdown
[百度][baidu]

[baidu]: https://www.baidu.com
```

---

### 6. 引用块

用 `>` 开头，可多行、可嵌套：

```markdown
> 这是一段引用。
> 可以多行写。

>> 嵌套引用
```

---

### 7. 行内代码与代码块

**行内代码**：用一对反引号 `` ` `` 包住：

```markdown
使用 `console.log()` 输出。
```

**代码块**：用三个反引号，并可标明语言：

````markdown
```javascript
function hello() {
  console.log("Hello");
}
```
````

**缩进代码块**（老式写法）：每行前加 4 个空格或一个 Tab。

---

### 8. 分隔线

单独一行，使用三个及以上的 `-`、`*` 或 `_`（中间可有空格）：

```markdown
---
***
___
```

---

### 9. 表格

```markdown
| 列1   | 列2   | 列3   |
|-------|-------|-------|
| 内容A | 内容B | 内容C |
| 左对齐 | 居中 | 右对齐 |
```

对齐方式（在分隔行用冒号）：

```markdown
| 左 | 中 | 右 |
|:---|:---:|---:|
| a  | b  | c  |
```

---

### 10. 转义字符

要显示字面意义的 `*`、`#`、`` ` `` 等，前面加反斜杠 `\`：

```markdown
\* 这不是列表项
\# 这不是标题
```

---

## 二、常见扩展（Obsidian / GitHub 等）

不同工具会在此基础上增加语法，例如：

| 语法 | 说明 |
|------|------|
| `[[笔记名]]` | Obsidian 双链 |
| `==高亮==` | 部分编辑器支持高亮 |
| `[^1]` 与文末 `[^1]: 注释` | 脚注 |
| `$$...$$` 或 `$...$` | 数学公式（LaTeX） |
| ` ```mermaid ` | 流程图、时序图等 |

学习时以你**实际用的软件**文档为准，核心仍是一节的通用语法。

## 三、如何更快学会 Markdown

### 1. 边用边记，不要一次背完

- 日常笔记、待办、README 都直接用 Markdown 写。
- 先掌握：**标题、列表、粗体/斜体、链接、代码块** 这五类，已覆盖 80% 场景。
- 遇到排版需求再查「表格、引用、任务列表」即可。

### 2. 准备一份自己的速查表

- 把本文或 [官方说明](https://www.markdownguide.org/) 里常用几条贴在笔记顶部。
- 在 Obsidian 里可设模板，新建笔记自动带上常用结构。

### 3. 对照预览学习

- Obsidian：编辑区与预览分屏（`Cmd/Ctrl + E` 切换预览）。
- VS Code：安装 Markdown 插件，侧边实时预览。
- 改一行、看一行效果，比纯看书快很多。

### 4. 刻意练习小练习（约 15 分钟）

用一篇笔记完成：

1. 写一个三级标题结构（章 / 节 / 小节）。
2. 列一个带勾选框的待办清单。
3. 贴一段带语言标识的代码块。
4. 做一张三列表格。
5. 加一条引用和一条外链。

练完一遍，基本语法就熟了。

### 5. 从模仿开始

- 打开仓库里已有的 `.md` 文件（如本库的 `README.md`），看源码再切预览。
- 复制别人的格式，改内容，比从零想语法省力。

### 6. 善用快捷键与插件

| 场景 | 建议 |
|------|------|
| Obsidian | 工具栏格式化、快捷键加粗/标题（可在设置里查看） |
| VS Code | `Markdown All in One` 等插件：目录、列表补全、预览 |
| 通用 | 学会 `` Ctrl/Cmd + ` `` 打行内代码、`**` 包粗体 |

### 7. 常见错误对照

| 现象 | 原因 | 改法 |
|------|------|------|
| 列表不显示 | `-` 后没空格 | 写成 `- 内容` |
| 标题无效 | `#` 后没空格 | 写成 `## 标题` |
| 代码块乱掉 | 反引号未成对 | 检查 ``` 是否闭合 |
| 表格错位 | 分隔行列数不一致 | 每行列数对齐 |

### 8. 推荐资源（按需查阅）

- [Markdown Guide（英文，结构清晰）](https://www.markdownguide.org/basic-syntax/)
- [GitHub Flavored Markdown 说明](https://github.github.com/gfm/)
- 你所用软件的帮助文档（如 Obsidian Help → Markdown）

## 四、最小可用模板（复制即用）

```markdown
# 笔记标题

> 摘要或引用一句重点。

## 要点

- [ ] 待办一
- [ ] 待办二

## 记录

正文内容，**重点**用粗体，`术语`用行内代码。

## 参考

- [链接名称](https://example.com)
```
